## Executive Summary

An end-to-end data science project on the [Brazilian Olist E-Commerce dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) — 100,000+ real orders placed between 2016 and 2018.

The project has two phases:
1. **EDA Phase** — Ingests 4 relational CSVs, merges them into a master dataset, and generates 3 business-driven visualizations covering revenue, category performance, and logistics.
2. **ML Phase** — Builds a binary classifier to predict whether an order will be delivered late, using engineered features from timestamps and product attributes.

## Phase 1: Business Questions (EDA)

1. **Revenue Trajectory** — How did Olist's monthly revenue trend across 2017–2018?
2. **Category Performance** — Which categories generate the highest revenue?
3. **Logistics Performance** — Which states have the worst average delivery times?

### Key Insights

**1. Revenue grew 3.4× between Jan–Nov 2017, driven by Q4 seasonality**
The monthly revenue time-series shows steep compound growth with a pronounced November spike (Black Friday). This confirms Olist was in rapid adoption phase — retention metrics matter more than GMV alone.

**2. Health & Beauty and Computer Accessories dominate revenue**
The top-2 categories significantly outpace categories ranked 3–10. This signals supplier concentration risk — diversifying into high-margin categories (watches, sports) is the lower-risk growth lever.

**3. Northern states (RR, AP, AM) average 2–3× the delivery time of SP**
São Paulo averages ~8 days. Roraima exceeds 25 days. This reflects last-mile carrier coverage gaps in low-density regions — not purely a distance problem.

## Phase 2: Late Delivery Prediction (ML)

### Problem Statement
Predict whether an order will be delivered late (`is_late = 1`) using features available at order placement time.

### Target Variable
```
is_late = 1  if actual delivery > estimated delivery date
is_late = 0  otherwise

Class distribution: 92.1% on-time | 7.9% late  (imbalanced)
```

### Features Engineered

| Feature | Description |
|---------|-------------|
| `approval_days` | Days from purchase → order approval |
| `estimated_delivery_window` | Days from purchase → estimated delivery |
| `purchase_month` | Month of purchase (seasonality) |
| `purchase_dayofweek` | Day of week 0–6 (weekend effects) |
| `product_volume_cm3` | Length × Height × Width |
| `price` | Order price |
| `freight_value` | Shipping cost |
| `product_weight_g` | Product weight |

### Model Results

| Model | Recall (Late) | Precision (Late) | ROC-AUC |
|-------|--------------|-----------------|---------|
| Logistic Regression (baseline) | 0.57 | 0.10 | 0.60 |
| **Random Forest** (`class_weight='balanced'`) | **0.27** | **0.51** | **0.79** |

> **Note on accuracy:** Overall accuracy is 92% for both models — but this is misleading on an imbalanced dataset (92:8 ratio). ROC-AUC and Late-class recall/precision are the meaningful metrics here.

> **Key finding:** `freight_value`, `price`, and `estimated_delivery_window` are the top 3 most important features. `approval_days`, despite seeming intuitive, has near-zero importance — data beats intuition.

## Reproducing This Analysis

### 1. Clone the repository
```bash
git clone https://github.com/Yash1bajpai/olist-eda.git
cd olist-eda
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Add the raw data
Download the Olist dataset from [Kaggle](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) and place all CSV files inside an `archive/` folder.

### 4. Run the pipeline
```bash
# Stage 1 — generates master_data.csv
python 1_data_preparation.py

# Stage 2 — generates EDA plots
python 2_eda_visualizations.py

# Stage 3 — trains models, saves to models/
python ml/4_model_training.py

# Stage 4 — generates evaluation plots
python ml/5_model_evaluation.py
```
